Challenge Overview
Wipro × AWS | Build Real Agentic AI Applications
Objective:
The AWS Codeathon challenges Wipro developers to build practical, high-impact Agentic AI applications using the full capabilities of Amazon Q Developer, Amazon Bedrock AgentCore, and complementary AWS services.
This initiative is jointly hosted by Wipro and AWS as part of the TopGear innovation ecosystem.
Goal:
Demonstrate how KIRO / Amazon Q Developer, Amazon Bedrock Agent Core, and AWS Transform work together to transform software development through intelligent, agent-driven automation.
Choose the theme that best fits your team's strengths and interests
Choose one of five themes to build your Agentic AI application. Each theme highlights AWS's AI-powered development ecosystem with a focus on multi-agent orchestration, spec-driven development, and intelligent automation, using industry-scalable use cases.
AWS Codethon - Challenge Themes.
Theme 1: AI-Powered Developer Productivity Platform - Detailed Use Cases.
Theme 2: Intelligent Customer Experience & Business Automation.
Theme 3: Intelligent Multi-Agent Domain Solutions.
Theme 4: Intelligent Data Analytics & AI Integration.
Theme 5: Legacy System Transformation with Intelligent Automation
AWS AI Services to Utilize:
• KIRO / Amazon Q Developer - AI-Driven Development (KIRO + Q → Build faster)
Purpose: Spec-driven, end-to-end software delivery
• KIRO IDE: Specs → guided → autonomous (autopilot) development with multi-file coordination and production focus
• KIRO CLI: Automates tasks, infra (IaC), CI/CD, and AWS operations via natural language
• Amazon Q Developer: AI coding, debugging, security scans, docs, and smart code suggestions inside IDEs
• Amazon Bedrock Agent Core - Multi-Agent Orchestration (Coordinate intelligent agents)
Purpose: Build and manage enterprise-grade agent systems
• Core: Secure runtime, tool/API gateway, identity & access control, shared memory, observability
• Collaboration: Supervisor-led agents with task delegation, aggregation, and monitoring
• AWS Transform - Legacy Modernization (Modernize legacy systems)
Purpose: Faster, automated modernization
• App discovery & assessment
• Automated code & DB transformation
• Monolith → microservices migration

## IntelPulse Submission Strategy

### Project Overview
**IntelPulse** is a production-grade threat intelligence platform migrating from VPS to AWS, demonstrating Theme 3 (Intelligent Multi-Agent Domain Solutions).

### Key Differentiators
1. **Multi-Agent Architecture**: Supervisor + 3 collaborators for IOC analysis
2. **Real-World Application**: 13 threat feeds, live IOC enrichment
3. **Production-Ready**: Enterprise security, 99.9% availability
4. **Complete AWS Integration**: Bedrock, ECS, OpenSearch, ElastiCache

### Evaluation Criteria Alignment

#### 1. Application Quality & Functionality (30 points)
- **Target**: 28/30
- **Strategy**:
  - Production-grade architecture (VPC, multi-AZ, security groups)
  - Comprehensive error handling (fallback chains)
  - Complete feature set (search, enrichment, analytics, reporting)
  - Load testing and monitoring

#### 2. Amazon Q Feature Utilization (30 points)
- **Target**: 30/30
- **Strategy**:
  - 16-task spec driving all development
  - 4 steering files (product, tech, aws-migration, coding-standards)
  - 3 agent hooks (test-sync, doc-update, security-scan)
  - KIRO CLI for AWS operations
  - Security scans with documented fixes
  - Detailed usage report with screenshots

#### 3. Productivity Demonstration (20 points)
- **Target**: 19/20
- **Strategy**:
  - 80% development time savings (75h → 15h)
  - Before/after metrics table
  - Quality improvements (code coverage, security issues)
  - Automation impact (feed triggers, IOC analysis)

#### 4. Innovation & Creativity (20 points)
- **Target**: 20/20
- **Strategy**:
  - Novel multi-agent pattern for threat intelligence
  - Lambda action groups for external API orchestration
  - Knowledge base integration (MITRE ATT&CK)
  - Reusable CDK constructs
  - Open source contribution potential
